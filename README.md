## 🧠 The AI Strategy

FINN uses a **3-layer intelligence system** for bulletproof performance:

### Layer 1 — Smart Cache ⚡
Every insight is hashed and cached for 24 hours. Identical data → instant response. No API calls. No latency.

### Layer 2 — Gemini 2.0 Flash 🚀  
Fresh insights powered by Google's fastest AI. Free tier. Production-ready. Generates personalized analysis in <2 seconds.

### Layer 3 — Rule-Based Fallback 🛡️
If Gemini fails (rate limits, network issues, API outage), our rule-based engine kicks in with **identical output structure**. Users never see a failure.

```
  User Request
       │
       ▼
  ┌────────────┐   HIT    ┌─────────────┐
  │   CACHE    │─────────▶│  Response   │
  └─────┬──────┘          └─────────────┘
        │ MISS
        ▼
  ┌────────────┐  SUCCESS ┌─────────────┐
  │   GEMINI   │─────────▶│  Response   │
  └─────┬──────┘          │   + Cache   │
        │ FAIL            └─────────────┘
        ▼
  ┌────────────┐          ┌─────────────┐
  │  FALLBACK  │─────────▶│  Response   │
  └────────────┘          └─────────────┘
```

**Result:** FINN never shows a failure. Judges never see a broken demo.

## 🔒 Security

FINN takes security seriously. **6 layers of protection:**

1. **🔐 Middleware Auth** — Protects all `/dashboard/*` and `/api/*` routes
2. **👤 Supabase Auth** — JWT + bcrypt + session management
3. **🛡️ Row Level Security** — Users only see their own data
4. **📄 File Validator** — PDF/CSV/XLSX only, 10MB max, safe filenames
5. **🚦 Rate Limiter** — 20/min chat, 10/min upload, 30/min insights
6. **🧹 XSS Sanitizer** — All user input sanitized, secrets server-only

## 📦 Getting Started

### Prerequisites
- Node.js 18+ 
- npm or yarn
- A [Supabase](https://supabase.com) account (free)
- A [Gemini API key](https://aistudio.google.com) (free)

### Installation

```bash
# 1. Clone the repo
git clone https://github.com/yourusername/finn.git
cd finn

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.example .env.local
```

### Environment Variables

Create `.env.local` with these values:

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_key

# Gemini AI
GEMINI_API_KEY=your_gemini_key

# App
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### Database Setup

```bash
# 1. Go to Supabase SQL Editor
# 2. Run the schema from /supabase/schema.sql
# 3. Enable Google + GitHub OAuth in Authentication → Providers
```

### Run Locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) and you're in! 🎉
