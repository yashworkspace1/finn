## рҹӨ– AI Architecture вҖ” 3-Model Fallback Chain

FINN never crashes due to rate limits. Every Gemini call tries 3 models in sequence:

```
Request вҶ’ gemini-2.5-flash
вҶ“ (rate limited?)
gemini-3-flash-preview
вҶ“ (rate limited?)
gemini-2.0-flash
вҶ“ (all failed?)
Local fallback memo
```

3 API keys Г— 3 models = **9 attempts before any failure**
Effective capacity: ~180 AI requests/day on free tier

## рҹҡҰ Getting Started

### Prerequisites

```bash
Node.js >= 18.0.0
npm >= 9.0.0
```

### 1. Clone the repository

```bash
git clone https://github.com/yourusername/finn.git
cd finn
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up environment variables

```bash
cp .env.example .env.local
```

Fill in your `.env.local`:

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# Gemini AI вҖ” add up to 3 keys for fallback
GEMINI_API_KEY_1=your_first_key
GEMINI_API_KEY_2=your_second_key
GEMINI_API_KEY_3=your_third_key
```

### 4. Set up Supabase database

Run the migration SQL in your Supabase SQL Editor:

```bash
# Copy contents of supabase/migrations/001_initial.sql
# Paste and run in Supabase SQL Editor
```

### 5. Run the development server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)
